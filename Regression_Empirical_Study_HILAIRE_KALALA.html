b) Categorical feature encoding

Categorical feature encoding is essential for building regression models because most machine learning algorithms, including linear regression, work with numerical data. Since categorical variables contain text or labels, they need to be converted into a numerical format that the model can interpret. Without encoding, the model cannot process these categorical features, leading to errors or inaccurate predictions.

To address this, I will use the One-Hot Encoding approach to transform categorical variables. This method converts each category of a variable into a binary column (0 or 1), ensuring that the model correctly interprets categorical information without assuming an ordinal relationship between categories.

First, I am going to transform the sex feature into 2 new features: male and female, then the region feature into the northeast, northwest, southeast, southwest features and finally, the smoker feature into a numerical feature (1 for yes, 0 for no).

c) EDA and Outlier detection

For this section, I am first going to program a LOF (Local Outlier Factor) approach to detect outliers. Second, I am going to use boxplot to visualize the distributions of data on the different numerical features to find a feature that contains outliers. Third, I am going to find the outliers using my LOF approach and remove them.

The Local Outlier Factor (LOF) is an unsupervised anomaly detection algorithm that identifies outliers by comparing the density of a given point to its neighbors. The key idea is that an outlier will have a much lower density compared to surrounding points. LOF assigns an anomaly score to each data point based on how isolated it is from its neighbors. The density is calculated by measuring the reachability distance between a point and its neighbors.

In[200]:
#Implementation of the LOF approach using the LocalOutlierFactor class from scikit-learn
from sklearn.neighbors import LocalOutlierFactor
def apply_Lof(feat):

    # Selecting the feat (feature) column for outlier detection
    feat_values = med_dt[[feat]]  # Must be in 2D format for LOF

    # Applying the Local Outlier Factor
    lof = LocalOutlierFactor(n_neighbors=50, contamination=0.01)  # 1% expected outliers
    outlier_labels = lof.fit_predict(feat_values)

    # LOF returns -1 for outliers, 1 for inliers
    med_dt['outlier_'+feat] = outlier_labels  # Adding results to dataset

    # Filtering out the outliers
    outliers = med_dt[med_dt['outlier_'+feat] == -1]

    #return the detected outliers
    print(outliers[[feat]])

d) Predictive analysis : Linear Regression (Explore the different LR classes suggested from Scikit-learn)

Linear regression attempts to model the relationship between two variables by fitting a linear equation ( Y = a.X + b) to observed data. One variable is considered to be an explanatory variable or an independent variable (X), and the other is considered to be a dependent variable (Y)[1]. Both simple and multiple linear regressions assume that there is a linear relationship between the input variable(s) and the output target variable. The main difference is the number of independent variables that they take as inputs. Simple linear regression just takes a single feature, while multiple linear regression takes multiple x values. The equation of a multiple linear regression looks like this Y = w0 + w1.X1 + w2.X2 + w3.X3 + ... + wn.Xn.

The whole point of a machine learning approach to train a linear regression model, is to determine the parameters called "weigthts" (wi) of the equation. With a training dataset, it starts by using randomly initialized values for each weight, impute the output using the linear function, then use a cost function that determine the loss of accuracy from the real output. So, we would want that cost function to be as minimized as possible, this is when the Gradient descent algorithm is useful. Without getting into it details, Gradient descent is a method of changing weights based on the loss function for each data point.

The Lasso class from sklearn.linear_model implements a linear regression based approach to build a predictive model. Its particularity is that it is useful when dealing with many irrelevant or highly correlated features. It provides automatic feature selection. I am going to use this class to do my regression empirical study for building a predictive model. I also found the ElasticNet class interesting beacause it actually keeps some correlated features to build the regression model while balancing sparsity and stability. But it is best for large datasets, the one I am working on is not large enough.

from sklearn.model_selection import KFold 

def build_eval_LassoCV(X_tr, y_tr):
    # Define K-Fold Cross-Validation (4 folds)
    kf = KFold(n_splits=4, shuffle=True, random_state=42) # I am going to use this class to split the training dataset in 4 "folds" (subsets)

    #The averaged evaluation metrics that will determine the performance of this system by aggregating the results of the 4 models that are going to be built
    mse_avg = 0.0
    r2_avg = 0.0


    # Splitting dataset into 4 subsets and store a tuple (tain_index, test_index) of each fold in a dictionnary object
    folds = [];
    y_np = np.array(y_tr)
    X_np = np.array(X_tr)

    for fold, (train_index, test_index) in enumerate(kf.split(X_np), 1):
        # Get the training set of the fold
        folds.append((train_index, test_index))

    # Now, I am going to make a while-loop that is going to use 3 folds (train_indexes only) to build a model and evaluate it on the 4th fold that wasn't used in the training,
    # then, it will aggregate the results of the evaluations to average them at the end.
    
    
    fld = 0
    while (fld <4):
        test = folds[fld]
        y_tst = y_np[test[1]] # I get the values of the target variable to perform the evaluation on the fldth fold (using the test_index)

        X_tr = np.array([], dtype=int)
        for i in range(4):
            if ( i!=fld):
                train = folds[i]
                X_tr = np.concatenate((X_tr, train[0]))
        # We are now training the model using 3/4 of the training rows 
        lasso = LassoCV(alphas=np.logspace(-3, 1, 100), cv=5)  # This will test multiple values for alpha and select the one that gives the best balance between accuracy and feature selection
        
        lasso.fit(X_np[X_tr], y_np[X_tr])

        # making the prediction
        y_pred = lasso.predict(X_np[test[1]])
        # Evaluate the Model
        mse_avg += mean_squared_error(y_tst, y_pred)
        r2_avg += r2_score(y_tst, y_pred)
        
        fld += 1
    

    rmse_avg = np.sqrt(mse_avg /4)
    r2_avg = r2_avg /4
    print(f"Averaged Root Mean Squared Error: {rmse_avg:.2f}")
    print(f"Averaged R Score: {r2_avg:.2f}")
    return rmse_avg, r2_avg, lasso
